Basic Example with Requests Library

This is the simplest way to use the proxy with Python's requests library:

import requests

# Proxy configuration
proxy_url = 'http://{{ proxy_config.username }}:{{ proxy_config.password }}@{{ proxy_config.host }}:{{ proxy_config.port }}'

proxies = {
    'http': proxy_url,
    'https': proxy_url
}

# Make a request
url = 'https://ipv4.icanhazip.com'
response = requests.get(url, proxies=proxies, timeout=10)

print(f"Your IP address: {response.text.strip()}")
print(f"Status code: {response.status_code}")

Complete Script with Error Handling

import requests

# Proxy credentials
PROXY_HOST = "{{ proxy_config.host }}"
PROXY_PORT = {{ proxy_config.port }}
PROXY_USER = "{{ proxy_config.username }}"
PROXY_PASS = "{{ proxy_config.password }}"

# Construct proxy URL
proxy_url = f'http://{PROXY_USER}:{PROXY_PASS}@{PROXY_HOST}:{PROXY_PORT}'
proxies = {
    'http': proxy_url,
    'https': proxy_url
}

def make_request(url, method='GET', **kwargs):
    """Make a request through the proxy"""
    try:
        if method.upper() == 'GET':
            response = requests.get(url, proxies=proxies, timeout=10, **kwargs)
        elif method.upper() == 'POST':
            response = requests.post(url, proxies=proxies, timeout=10, **kwargs)
        else:
            raise ValueError(f"Unsupported method: {method}")
        
        response.raise_for_status()
        return response
    
    except requests.exceptions.ProxyError as e:
        print(f"Proxy Error: {e}")
        return None
    except requests.exceptions.RequestException as e:
        print(f"Request Error: {e}")
        return None

# Example usage
if __name__ == "__main__":
    # Test the proxy
    response = make_request('https://ipv4.icanhazip.com')
    if response:
        print(f"‚úÖ Success! Your proxy IP: {response.text.strip()}")
    
    # Make a GET request to an API
    response = make_request('https://api.github.com/users/github')
    if response:
        print(f"‚úÖ API Response: {response.status_code}")
        print(response.json())

Using with Sessions (Recommended for Multiple Requests)

import requests

# Create a session with proxy configuration
session = requests.Session()
proxy_url = 'http://{{ proxy_config.username }}:{{ proxy_config.password }}@{{ proxy_config.host }}:{{ proxy_config.port }}'
session.proxies = {
    'http': proxy_url,
    'https': proxy_url
}

# Make multiple requests using the same session
urls = [
    'https://ipv4.icanhazip.com',
    'https://api.ipify.org',
    'https://ifconfig.me/ip'
]

for url in urls:
    try:
        response = session.get(url, timeout=10)
        print(f"IP from {url}: {response.text.strip()}")
    except Exception as e:
        print(f"Error: {e}")

JavaScript/Node.js (using axios)

const HttpsProxyAgent = require('https-proxy-agent');

.catch(error => {
    console.error('Error:', error.message);
});

üí° Best Practices

‚úÖ DO:
  • Use sessions for multiple requests to improve performance
  • Implement proper error handling for proxy failures
  • Set reasonable timeouts (10-30 seconds recommended)
  • Keep your credentials secure and never commit them to public repositories
  • Use environment variables for credentials in production

üÜò Troubleshooting

Common Issues and Solutions

Problem: ProxyError or Connection Timeout
Solution:
  • Check your internet connection
  • Verify credentials are correct
  • Increase timeout value (e.g., timeout=30)
  • Ensure firewall isn't blocking outbound connections
Problem: Authentication Failed (407 Error)
Solution:
  • Double-check username and password
  • Make sure password includes the country specifier: _country-us
  • Verify credentials haven't been changed
Problem: SSL Certificate Errors
Solution:
  • Update your SSL certificates: pip install --upgrade certifi
  • As a last resort (not recommended for production): verify=False
