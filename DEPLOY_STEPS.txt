Run these commands in your terminal:

================================================================================
                    STEP 2: DEPLOY ON DIGITALOCEAN
================================================================================

1. Go to DigitalOcean App Platform:
   https://cloud.digitalocean.com/apps

2. Click "Create App"

3. Choose "GitHub" as source

4. Authorize DigitalOcean to access your GitHub (if first time)

5. Select Repository:
   - Repository: bbarnes4318/proxy
   - Branch: main
   - Check "Autodeploy" (deploys automatically on push)

6. DigitalOcean will detect the .do/app.yaml configuration automatically!
   You should see:
   - Service Type: Web Service
   - Environment: Python
   - Run Command: gunicorn --worker-tmp-dir /dev/shm --config gunicorn_config.py app:app

7. Choose Resources:
   - Basic Plan: $5/month (512MB RAM) - RECOMMENDED
   - Professional: $12/month (1GB RAM)

8. Choose Region:
   - Select closest to your users (e.g., New York for US East)

9. Review and Create:
   - App name: proxy-access-portal (or customize)
   - Review all settings
   - Click "Create Resources"

10. Wait for Deployment (3-5 minutes)
    - You'll see build logs in real-time
    - Wait for "Deployed successfully" message

11. Access Your App:
    - You'll get a URL like: https://proxy-access-portal-xxxxx.ondigitalocean.app
    - Click it to open your app!

================================================================================
                        TROUBLESHOOTING
================================================================================

Problem: Git push fails
Solution: Make sure you're authenticated with GitHub
          Use --force flag if repository exists

Problem: DigitalOcean build fails
Solution: Check build logs in dashboard
          Verify requirements.txt is correct
          Ensure Python version matches runtime.txt

Problem: App starts but shows errors
Solution: Check runtime logs
          Verify proxy credentials are correct
          Check environment variables

Problem: Can't access the app
Solution: Wait 5 minutes for deployment to complete
          Check that app is "Running" in dashboard
          Try the URL in a different browser
